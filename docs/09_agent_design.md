# Agent 设计说明

## 目标
- 让助理可以自动拆解任务、调用工具、生成最终结果。

## 多步骤流程（示例）
1. 识别意图：包含“报销总额 + 邮件草稿”。
2. 工具调用：`get_reimbursement_summary` 查询明细。
3. 生成结果：汇总金额与明细，输出邮件草稿。

## 工具选择策略
- 涉及报销金额/人员/进度：强制调用 MCP。
- 规则/制度类问题：优先走 RAG，附引用来源。

## 错误与兜底
- 若工具返回为空：提示用户补充姓名/月份。
- 若 RAG 召回不足：提示用户改写问题或补充关键词。

## 调用证据
- 运行日志：`outputs/agent_demo.log`
