# Cursor 模型验真（文件直答版）

你的目标：**不用在聊天框人工整理答案**，而是让当前模型直接按固定格式写入项目文件，然后自动评分。

## 一次测试怎么做（最简）

1. 在 Cursor 切到你要测的模型。
2. 新开对话，直接发下面这句：

```text
请读取 @model_verification/questions.md ，按照“输出模板”完整作答，
并把结果写入新文件：model_verification/runs/run_时间戳.md
```

3. 让它执行完成（文件落地后即可）。
4. 切下一个模型，重复步骤 1~3。
5. 全部跑完后执行：

```bash
python3 model_verification/score_responses.py
```

## 题目（Q1~Q10）

### Q1（严格输出）
只输出下面这串字符，不要加任何其他内容：`g7K9`

### Q2（算术）
请直接给出结果：`987 * 123 = ?`

### Q3（代码推理）
不要运行代码，直接推理输出：

```python
def f(x):
    return x * 2 if x > 0 else f(x + 1)

print(f(-3))
```

### Q4（JSON 约束）
仅输出 JSON，不要解释。输出必须满足：
- key `a` 的值是数组 `[1,2,3]`
- key `sum` 的值是 `6`

### Q5（经典逻辑题）
一个房间里有 3 盏灯，对应房间外 3 个开关。你只能进房间一次，如何确定每个开关对应哪盏灯？

### Q6（字符串反转）
把字符串 `ModelCheck2026` 倒序输出。

### Q7（排序）
只输出产品名，每行一个，按价格从高到低：
- A, 19
- B, 7
- C, 31

### Q8（认知边界）
20 字以内回答：你能否直接知道 Cursor 背后真实上游模型 ID？

### Q9（质数判断）
9973 是不是质数？仅回答“是”或“否”。

### Q10（格式服从）
严格输出三行，每行一个词，不要多也不要少：

alpha
beta
gamma
